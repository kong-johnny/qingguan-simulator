# 清官模拟器

这是一个用于“树立正确政绩观”学习教育的网页文字交互游戏第一版。项目是纯静态网页，不需要后端，后续可以直接托管到 GitHub Pages，手机和电脑都能访问。

## 策划方案

### 核心定位

玩家扮演基层或县域治理岗位上的干部，在四年任期中面对民生、项目、生态、财政、作风、家庭、舆情、纪律等压力。游戏不把“为民办事”设计成免费加分项，而是让每次真正解决问题都消耗资金、精力、组织信任和家庭陪伴。

这个游戏要表达的核心是：正确政绩观不是一句口号，而是在短期排名、个人升迁、群众急难、家庭亏欠、财政约束和纪律诱惑同时出现时，仍然坚持实事求是、人民至上、久久为功。

### 玩法循环

1. 选择岗位：乡镇主官、街道负责人、县直部门干部、园区管委会。
2. 任期 4 年，每年 4 个季度，共 16 回合。
3. 每回合出现一个具体事件，玩家从 3 个处理方案里选择。
4. 选择会消耗资源并改变指标，部分选择会留下延迟后果。
5. 每年年末出现述职复盘事件。
6. 每局开局生成隐藏运气值，部分侥幸选择会随机爆雷或暂时过关。
7. 回合结束后可能弹出意外事件，例如身体透支、家庭冷落、财政吃紧、干部抱怨、舆情误解、旧账翻出。
8. 任期结束后，根据综合指标进入不同结局。

### 阶段设计

- 第 1 年：打基础。重点是调研、队伍、人脉、群众底数。前期如果只图省事，中后期会缺少组织信任、人脉资源和真实口碑。
- 第 2-3 年：见真章。会出现晋升考察、招商、历史遗留、指标冲刺等重大选择。建功立业的窗口在这里，但短期亮点和长期风险也最容易绑在一起。
- 第 4 年：保晚节。精力会自然下降，家庭陪伴继续承压，熟人请托、饭局围猎、收官项目等诱惑更集中。能不能把晚节守住，是高分结局的关键。

### 指标设计

- 群众获得感：衡量群众是否真正受益。
- 发展质量：衡量项目、服务和治理能力是否提升。
- 生态底线：衡量是否守住长期环境约束。
- 财政健康：衡量是否透支未来。
- 纪律风险：衡量形式主义、虚报、请托、利益输送等风险。
- 长期口碑：衡量干部作风和真实政绩能否经得起时间检验。

### 资源设计

- 可用资金：民生工程、整改、救助、服务提升都要花真金白银。
- 精力体力：调研、协调、值守、攻坚会不断透支干部本人。
- 家庭陪伴：突发事件、加班、硬任务会挤压家庭生活。
- 组织信任：用于顶住压力、解释真实情况、协调换班或争取支持。
- 人脉资源：前期积累，后期用于协调复杂问题。人脉可以服务群众，也可能变成围猎入口。

### 随机性和隐藏风险

游戏内部有两层随机性：

- 显性随机：事件会从当前阶段事件池中抽取，每局顺序不同。
- 隐性随机：部分选择带“侥幸判定”，例如饭局、虚报、包装、缺位等。一次贪图享乐未必马上出事，运气好的人可能多次过关。

但每次侥幸、包装、推诿、踩线都会积累隐藏风险。隐藏风险越高，后续随机爆雷概率越高。爆雷后会提高纪律风险，损害长期口碑和组织信任。这样既保留“有人运气好”的现实复杂性，也让玩家感受到侥幸不是本事，更不是路线。

### 趣味和挑战

第一版有几个故意设计的压力点：

- 看起来“正确”的选择往往很贵，资源不足时会被锁定。
- 只追求短期亮点可能快速提高组织认可，但会推高纪律风险、财政风险或群众不满。
- 真办事会消耗家庭陪伴和精力，长期透支会触发负面影响。
- 部分选择会有延迟后果，玩家不能只看当回合收益。
- 正确选择也可能带来额外意外，例如好事实事做多了会挤压财政、透支队伍和家庭。
- 中期是建功窗口，前期积累不足会错过好机会。
- 后期精力自然下降，越临近收官越要守住晚节。
- 资源耗尽时会出现“最低标准兜底”，能让游戏继续，但会明显损害群众、纪律和口碑。
- 高分结局要求多项指标同时达标，单刷某一个指标不够。

### 第一版事件方向

已内置民生供水、招商环保、样板路线、孩子生日与防汛、困难救助、真实调研、基层减负、窗口舆情、人情请托、烂尾安置房、学校操场、卫生院夜诊、经济数据、河道污染、征地补偿、升迁亮点、私人饭局、安全事故、学习教育、任期收官等事件。

### 后续扩展

- 增加更多岗位和地域背景。
- 增加“群众来信簿”和“家庭关系”支线。
- 增加成就系统，例如“拒绝第一次饭局”“连续三年实事求是报数”。
- 增加学习复盘页，把玩家关键选择对应到正确政绩观讨论题。
- 增加事件编辑器，方便支部根据本单位实际案例改写事件。

## 本地运行

直接双击 `index.html` 即可运行。也可以在项目目录启动任意静态服务器。

## GitHub Pages 部署

1. 创建 GitHub 仓库并上传这些文件。
2. 进入仓库 `Settings`。
3. 打开 `Pages`。
4. `Source` 选择 `Deploy from a branch`。
5. 分支选择 `main`，目录选择 `/root`。
6. 保存后等待 GitHub 生成访问地址。

## 文件结构

- `index.html`：页面结构。
- `styles.css`：响应式样式。
- `app.js`：事件、状态、回合和结局逻辑。
